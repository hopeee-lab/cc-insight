## 1. 产品定位

CC Insight 是一个本地 CLI 工具，读取 `~/.claude/` 目录数据，在浏览器中生成 Claude Code 个人使用画像。

**目标用户：** Claude Code 用户（开发者）
**发布路径：** 先自用 → 开源 GitHub → 视反响考虑商业化
**差异化：** `/insights` 输出文字报告；CC Insight 是实时可视化仪表盘 + 历史趋势 + Skill 管理

---

## 2. 技术架构

### 2.1 产品形态

- **类型：** 本地 CLI 工具
- **安装：** `npm install -g cc-insight` 或 `brew install cc-insight`
- **启动：** 终端运行 `cc-insight`，自动启动本地 HTTP 服务并打开浏览器
- **停止：** 关闭终端进程，服务停止，无后台驻留

### 2.2 数据层：SQLite 索引

- 首次启动扫描 Claude 数据目录全量建库，后续增量更新
- 首次建库时浏览器显示进度条（"正在建立索引 xx%"），完成后自动进入 Dashboard；后续每次打开秒开
- 数据库路径：`~/.cc-insight/data.db`（用户无感）
- `fs.watch` 监听目录变化，触发增量解析
- WebSocket 推送前端实时刷新
- 数据库表预留 `source` 字段，支持未来多 CLI 工具扩展

### 2.3 核心表结构

```sql
-- session 记录，预留 source 字段
sessions (
  id, source, start_time, end_time, duration_sec,
  project_path, message_count, tool_use_count
)

-- skill / agent / plugin 注册表
tools (
  id, name, type, subtype, description,
  source_type, source_url,
  installed_at, updated_at,
  security_scan_result
)

-- 工具调用记录
tool_invocations (
  id, session_id, tool_name, invoked_at
)
```

### 2.4 产品大小估算

| 内容 | 大小 |
|------|------|
| 工具本体（Node.js + 依赖） | 5–15 MB |
| 数据库（轻度用户） | 1–5 MB |
| 数据库（重度用户/年） | 50–100 MB |

## 3. 数据源

### 3.1 Claude 数据目录

CC Insight 一期同时支持 **Claude Code CLI** 和 **Claude 桌面 App**，两者数据格式相同（JSONL），合并统计，不区分来源。

| 来源 | 路径 |
|------|------|
| Claude Code CLI | `~/.claude/`（默认） |
| Claude 桌面 App（macOS） | `~/Library/Application Support/Claude/claude-code-sessions/` |

CLI 路径支持通过环境变量覆盖：
```bash
CLAUDE_DIR=/custom/path cc-insight
```

启动时按以下优先级确定 CLI 目录：
1. 环境变量 `CLAUDE_DIR`
2. 默认 `~/.claude/`

桌面 App 目录自动检测，存在则合并扫描，不存在则跳过（跨平台兼容）。

### 3.2 数据文件

| 数据 | 相对路径 | 说明 |
|------|------|------|
| Skill 列表 | `skills/*/SKILL.md` | 名称、描述、来源 |
| 项目历史 | `projects/**/*.jsonl` | 对话记录、tool 调用、时间戳 |
| 全局配置 | `settings.json` | MCP server、全局配置 |
| Plugin 列表 | `plugins/` | 已安装插件目录 |

## 5. 一期功能模块

### 5.1 主题 1：使用概览

**布局：**

```
[ SESSIONS ] [ DURATION ] [ PEAK HOUR ] [ SILENT DAYS ]

┌─────────────────┬──────────────────────────────────┐
│                 │   ACTIVITY HEATMAP                │
│   INSIGHTS      ├──────────────────────────────────┤
│                 │   24H 时间分布                    │
└─────────────────┴──────────────────────────────────┘
```

**时间范围筛选：** 默认 7天，可选 30天 / 90天 / 全部（全局控制，影响两个主题所有数据）

**4 个指标卡片：**

| 指标 | 说明 |
|------|------|
| SESSIONS | 时间范围内对话次数 |
| DURATION | 累计使用时长 |
| PEAK PERIOD | 最活跃时间段，格式 `hh:mm–hh:mm` |
| AVG DURATION | 日均使用时长 |

**INSIGHTS 洞察卡片（动态生成）：**

基于用户实际数据自动计算，有数据才显示对应洞察，不写死固定条目。可能包含的洞察类型：
- 最高产的一天（当期最长使用时长 + session 数）
- 最长静默期（连续未使用天数，需达到一定阈值才展示）
- 时间习惯（夜猫子 / 早鸟 / 上班族，基于 24H 分布自动判断）
- 使用趋势（与上一个同等时间段对比，有明显变化才展示）
- 趣味数据（最短 / 最长 session，数据足够时展示）

每条洞察均附数据来源，不展示无数据支撑的结论。

**ACTIVITY HEATMAP：** GitHub contribution graph 风格，按日期展示活跃度

**24H 时间分布：** 柱状图，高亮峰值时段，标注静默区间

---

### 5.2 主题 2：Skill & Agent & Plugin

**布局：**

```
[ Skill 使用率 ] [ Agent 使用率 ] [ Plugin 使用率 ]

┌──────────────────┬────────────────────────────────┐
│  最常用（Top 5） │                                │
│                  │   全部列表                      │
│  从未使用        │   （可按类型 / 吃灰筛选）        │
│  （有则显示）    │                                │
│                  │                                │
│  RECOMMENDATIONS │                                │
└──────────────────┴────────────────────────────────┘
```

**时间范围筛选：** 默认 7天，可选 30天 / 90天 / 全部（与主题 1 共享同一全局筛选）

**3 个概览卡片（含进度条）：**
- 安装总数
- 使用率（已使用 / 总数，如 21/27）
- 从未使用数量

**最常用 Top 5：** 时间范围内调用次数排行，标签云形式

**从未使用：** 仅在存在时显示，列出名称 + 安装天数

**全部列表（每条记录）：**

| 字段 | 说明 |
|------|------|
| 名称 | - |
| 类型 | Skill / Agent / Plugin |
| 描述与用途 | 来自 SKILL.md |
| 来源类型 | 自行开发 / 下载 |
| 来源链接 | 下载时附原始 URL |
| 安装时间 | - |
| 更新时间 | - |
| 安全审查 | ✓ 安全 / ⚠️ 警告 / 未审查；首次启动时静态扫描 SKILL.md，结果缓存到 DB |
| AI 推荐 | 基于使用频率 + SKILL.md 描述自动生成建议（v1 不依赖外部用户 profile） |
| 使用次数 | - |
| 上次使用 | - |
| 吃灰标记 | 超过 30 天未使用标红 + 降低透明度（默认阈值 30 天，未来可配置） |
| 删除 | 单条删除按钮 |

**列表筛选：** 全部 / Skill / Agent / Plugin / 吃灰

**RECOMMENDATIONS：**
- 建议安装：根据用户背景推荐未安装的工具
- 建议清理：吃灰工具一键批量清理入口

## 6. 交互说明

| 功能 | 说明 |
|------|------|
| 实时刷新 | 检测到数据变化时浏览器自动更新，无需手动刷新 |
| 单条删除 | 列表每条记录右侧删除按钮 |
| 批量清理 | RECOMMENDATIONS 区域一键清理所有吃灰工具 |
| 主题切换 | 深色 / 浅色 / 跟随系统，状态持久化 |
| 时间筛选 | 全局生效，切换后两个主题数据同步更新 |

## 7. UI 规范

- **最小字号：** 14px，全页面强制执行，不得低于此值
- **字体：** monospace 为主
- **深色主题配色：** `#0d1117` 背景，`#4ade80` / `#22d3ee` accent
- **动画：** 数字滚动加载、卡片渐入，轻量不花哨
