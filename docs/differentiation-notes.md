# CC Insight vs /insights 差异化定位

> 用于写入 README，说明 CC Insight 的独特价值

## /insights（Claude Code 内置）
- 输出形式：纯文本报告，一次性生成
- 本质：LLM 读对话内容，写诊断报告
- 局限：主观、不可横向比较、不随时间更新

## CC Insight Topics（可视化画像）
- 输出形式：可视化 + 可交互 + 持续更新
- 量化占比：调试占 38% 时间，一眼可见
- 趋势对比：本周 vs 上周，话题结构变化（二期）
- 高频词云：每类话题下的具体技术词
- 时间筛选：7d/30d/90d，随主筛选器联动

## 核心价值主张
> "你以为在探索，其实 60% 的时间在调 bug"
> 用数据照镜子，不是读报告。

本质区别：/insights 是 LLM 的主观感知，CC Insight 是客观数据驱动的工作画像。
